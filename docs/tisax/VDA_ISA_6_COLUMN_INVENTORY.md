## Key Finding: Column Q Contains Two Extractable Frameworks

Column Q "Verweisung auf Implementierungsanleitung" contains three sub-sections
per cell, delimited by blank lines:

```
BSI-Standard 200-2:
  3.2, 3.4          <-- section references, NOT extracted (ambiguous format)

BSI IT-Grundschutz-Compendium:
  ISMS.1, ORP.1     <-- ANCHOR IDs extracted -> tisax-vda-isa-6_to_bsi-grundschutz_v1.0.yaml

NIST SP800-53r5:
  PT-1, AC-1, AT-1  <-- ANCHOR IDs extracted -> tisax-vda-isa-6_to_nist-sp800-53r5_v1.0.yaml
```

The BSI-Standard 200-2 numeric section refs (`3.2`, `4.1.6`, `10.1.4`) match
the same regex as VDA-ISA control IDs and cannot be safely disambiguated
programmatically — they are intentionally skipped.

## GDPR References in DP Sheet

The Datenschutz sheet contains GDPR article references, but they are
embedded in narrative requirement text in col J, not in a standalone anchor column.
Example: `"Art. 37 DSGVO — Bestellung eines Datenschutzbeauftragten..."`.
These are ENX-licensed content and are NOT extracted.

If GDPR <-> VDA-ISA mapping is needed, derive it transitively via
ISO 27701 (privacy extension to ISO 27001) using the existing
`tisax-vda-isa-6_to_iso27001-2022_v1.0.yaml` bridge.
