Mood-Whitelist (Single-Source-of-Truth)

Alva-Mood-Werte sind geschlossen. Nur die folgenden 9 Moods werden vom Renderer (assets/components/AlvaCharacter.jsx, AlvaMoodExtension) unterstützt:

idle thinking working happy sleeping warning focused scanning celebrating

Fallback-Verhalten: Nicht-gelistete Mood-Werte (z.B. pointing, curious, excited) werden silently auf idle degradiert — keine Konsolen-Warnung, kein Render-Fehler. Folge: ein Tippfehler ist ein lautloser Regression-Bug. Bei jedem neuen Hint/EmptyState/Hero: aus dieser Liste auswählen, sonst läuft Alva mit dem falschen Mood. Anti-Pattern siehe DESIGN_SYSTEM_CONFORMANCE.md Issue 4 + A4.

Alva

Alva — die Companion-Fee

Alva ist der Charakter, der das System menschlicher macht — ohne infantil zu werden. Sie hat einen hellen Körper mit dunklen Face-Features in beiden Modi (Body invertiert nicht mit Dark-Mode). Flügel und Aura folgen --primary / --accent; pro Mood verschiebt sich der Aura-Tint.

Anatomie

  • Body + Kopf#f4f7ff (light) · #e2e8f0 (dark) — immer hell
  • Face-Features#1e1b4b (light) · #0a0e1a (dark) — immer dunkel
  • Flügel-Tint links= --primary, mit accent-Verlauf
  • Flügel-Tint rechts= --primary, mit accent-Verlauf
  • Auraradialer Glow, mood-abhängiger Tint & Skala
  • Krone= --accent, drei Punkte am Haaransatz — immer da
  • Wand= --accent, im rechten Arm — verschwindet beim Sleeping-Mood
Nie: Alvas Body auf var(--fg) oder var(--ink) setzen — das invertiert mit Dark-Mode und macht sie schwarz. Body bleibt immer in der hellen Familie (#f4f7ff / #e2e8f0).

Moods

Neun Mood-States. Alva wechselt kontextabhängig — nie zufällig. Jeder Mood hat Aura-Tint, Body-Animation, Flügelschlag-Tempo und mood-spezifische Deko (Z-Z-Z, Konfetti, Reticle, Cogs). Der Mood ist ein Prop: <FairyCharacter mood="thinking" />.

import { FairyCharacter } from './states/FairyCharacter';

    accent:  'var(--accent)',
    aura:    'var(--aura)',
};



// Mögliche Moods:

// warning · focused · scanning · celebrating`}

Alva · Hint-DTO

AlvaHint — priorityTier vs. variant

Im AlvaHint-DTO (src/AlvaHint/AlvaHint.php) sind Tier und Variant zwei separate Properties — kein Schreib-Alias füreinander. Sie haben unterschiedliche Bedeutung und werden parallel gesetzt.

Property-Definition

  • priorityTier: intErlaubte Werte: 1 · 2 · 3. Constructor wirft InvalidArgumentException bei anderen Werten.
  • variant: stringErlaubte Werte: 'info' · 'warning' · 'danger' · 'success'. Steuert Aurora-Tone (siehe _fa_alert).
  • dismissible: boolTier 1 erzwingt dismissible: false — Constructor blockt sonst. Eine 72h-DSGVO-Frist darf nicht weggeklickt werden.
  • mood: stringAlva-Mood aus der Whitelist oben (default 'thinking').

Tier ↔ Variant Mapping (Konvention)

Tier Bedeutung Default-Variant Dismissible Aurora-Frame
1 · Pflicht (regulatory) blocking · high-prio · z.B. DSGVO 72h-Frist, abgelaufene CAPA danger (oder warning) erzwungen nein roter Aurora-Frame · --danger
2 · Audit-Gap (warn) warn · z.B. fehlende Owner, fehlende Severity-Definition warning erlaubt orange Aurora-Frame · --warning
3 · Efficiency (info) info · Tipp · z.B. Bulk-Import-Suggestion, Workflow-Optimierung info (oder success) erlaubt neutral · --primary 
// ✅ Tier 1 · Regulatory · zwingt dismissible=false
return new AlvaHint(
    key:                       'data_breach.deadline_72h',
    titleTranslationKey:       'alva.hint.data_breach.title',
    bodyTranslationKey:        'alva.hint.data_breach.body',
    translationDomain:         'alva',
    variant:                   'danger',       // ← Aurora-Tone
    priorityTier:              1,              // ← Schweregrad
    dismissible:               false,          // ← Tier 1 erzwingt false
    mood:                      'warning',      // ← aus Mood-Whitelist
    entityType:                'DataBreach',
);

// ✅ Tier 2 · Audit-Gap
return new AlvaHint(
    key:                       'risk.missing_owner',
    titleTranslationKey:       'alva.hint.risk.no_owner.title',
    bodyTranslationKey:        'alva.hint.risk.no_owner.body',
    variant:                   'warning',
    priorityTier:              2,
    dismissible:               true,
    mood:                      'thinking',
);

// ✅ Tier 3 · Efficiency-Tipp
return new AlvaHint(
    key:                       'risk.bulk_import_tip',
    titleTranslationKey:       'alva.hint.risk.bulk_tip.title',
    bodyTranslationKey:        'alva.hint.risk.bulk_tip.body',
    variant:                   'info',
    priorityTier:              3,
    dismissible:               true,
    mood:                      'happy',
);

// ❌ Anti-Pattern · Tier als String
new AlvaHint(... priorityTier: 'high' ...);   // TypeError
new AlvaHint(... priorityTier: 1, dismissible: true ...);   // InvalidArgumentException
Anti-Pattern (A10): Tier-Wert als String ('high', 'medium') übergeben oder Variant über Tier abgeleitet rendern (variant ≡ tierToColor(tier)). Solutions-Dokumente haben das mehrfach so gezeigt — nicht ausführen. Tier und Variant sind unabhängig und werden beide explizit gesetzt.
