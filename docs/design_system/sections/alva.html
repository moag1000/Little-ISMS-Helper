Alva

Alva — die Companion-Fee

Alva ist der Charakter, der das System menschlicher macht — ohne infantil zu werden. Sie hat einen hellen Körper mit dunklen Face-Features in beiden Modi (Body invertiert nicht mit Dark-Mode). Flügel und Aura folgen --primary / --accent; pro Mood verschiebt sich der Aura-Tint.

Anatomie

  • Body + Kopf#f4f7ff (light) · #e2e8f0 (dark) — immer hell
  • Face-Features#1e1b4b (light) · #0a0e1a (dark) — immer dunkel
  • Flügel-Tint links= --primary, mit accent-Verlauf
  • Flügel-Tint rechts= --primary, mit accent-Verlauf
  • Auraradialer Glow, mood-abhängiger Tint & Skala
  • Krone= --accent, drei Punkte am Haaransatz — immer da
  • Wand= --accent, im rechten Arm — verschwindet beim Sleeping-Mood
Nie: Alvas Body auf var(--fg) oder var(--ink) setzen — das invertiert mit Dark-Mode und macht sie schwarz. Body bleibt immer in der hellen Familie (#f4f7ff / #e2e8f0).

Moods

Neun Mood-States. Alva wechselt kontextabhängig — nie zufällig. Jeder Mood hat Aura-Tint, Body-Animation, Flügelschlag-Tempo und mood-spezifische Deko (Z-Z-Z, Konfetti, Reticle, Cogs). Der Mood ist ein Prop: <FairyCharacter mood="thinking" />. 

{`// React-Komponente
import { FairyCharacter } from './states/FairyCharacter';

const tokens = {
    primary: 'var(--primary)',
    accent:  'var(--accent)',
    aura:    'var(--aura)',
};



// Mögliche Moods:
// idle · thinking · working · happy · sleeping
// warning · focused · scanning · celebrating`}

Mood-Matrix

  • idleDefault · Dock · Header · ruhiges Atmen, langsames Blinzeln
  • thinkingKI-Vorschlag · Form-Suggestion · Kopfneigung + ?-Bubble
  • workingBatch-Import · Long-Running-Task · zwei rotierende Cogs
  • happyForm gespeichert · Task abgeschlossen · ✦-Funken + Bounce
  • sleepingArchiv · leerer Filter · z-z-z-Animation, Wand verschwindet
  • warningReview fällig · Kontroll-Frist · gelbes ⚠ + Shake
  • focusedAudit-Mode · Drill-Down · rotierendes Reticle um den Body
  • scanningRisk-Scan · Asset-Discovery · Cyan-Scanline läuft durchs Frame
  • celebratingISO-Cert erreicht · Audit bestanden · Konfetti + schneller Flap

Dock & Placements

Alva hat fünf definierte Platzierungen — und nur diese. Sie erscheint nie „einfach so" in der UI.

  • dockfixed bottom-right · 24px / 72px Offset · z-9500 · Mood folgt App-State
  • empty-statezentriert, ~96px · über dem Title · idle/sleeping/curious
  • form-hint16×16px · neben Input-Label · thinking → suggestion
  • alert-inline24×24px · links im Alert · warning für eigene Trigger
  • hero128px · Login/Onboarding · einzige Stelle für volle Größe

Dock-Tokens

  • --alva-dock-offset-x24px — Abstand zum rechten Rand
  • --alva-dock-offset-y72px — Platz für Footer + Symfony-Dev-Toolbar
  • --alva-z9500 — über allem außer Modals/Toasts
Nie: Alva als reine Deko in Footer, Marketing-Banner oder Login-Hintergrund. Wenn sie da ist, ist sie aktiv — der Mood passt zum Screen-Kontext.
