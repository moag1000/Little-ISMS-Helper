# BSI-Standard 200-2: IT-Grundschutz-Methodik

## Ueberblick

BSI 200-2 beschreibt die Methodik des IT-Grundschutzes. Kern ist die systematische Absicherung von Informationsverbuenden durch Zuordnung von Bausteinen aus dem IT-Grundschutz-Kompendium.

## Drei Absicherungsstufen

### Basis-Absicherung
- **Ziel**: Breiter Einstieg in den IT-Grundschutz
- **Umfang**: Nur Basis-Anforderungen (B) aller relevanten Bausteine
- **Zielgruppe**: KMU, Einsteiger, schneller Ueberblick
- **Aufwand**: Gering
- **Zertifizierbar**: Nein (kein BSI-Zertifikat, aber Audit moeglich)

### Standard-Absicherung
- **Ziel**: Vollstaendiger IT-Grundschutz
- **Umfang**: Basis (B) + Standard (S) Anforderungen aller Bausteine
- **Zielgruppe**: Organisationen mit normalem Schutzbedarf
- **Aufwand**: Mittel bis hoch
- **Zertifizierbar**: Ja (ISO 27001 auf Basis IT-Grundschutz)

### Kern-Absicherung
- **Ziel**: Schutz der "Kronjuwelen"
- **Umfang**: Basis + Standard + Hoch (H) fuer kritische Assets
- **Zielgruppe**: Besonders schutzwuerdige Bereiche
- **Aufwand**: Hoch (aber gezielt)
- **Zertifizierbar**: Ja (fuer definierten Scope)

## Vorgehensweise

### Phase 1: Strukturanalyse
1. **Informationsverbund definieren** (Scope)
2. **Geschaeftsprozesse erfassen**
3. **IT-Systeme erheben** (Server, Clients, Netzkomponenten)
4. **Anwendungen dokumentieren**
5. **Raeumlichkeiten erfassen**
6. **Netzplan erstellen**
7. **Gruppenbildung** (gleichartige Objekte zusammenfassen)

### Phase 2: Schutzbedarfsfeststellung
- **Schutzziele**: Vertraulichkeit, Integritaet, Verfuegbarkeit
- **Schutzbedarfskategorien**:

| Kategorie | Definition | Schadenshoehe |
|-----------|-----------|---------------|
| **Normal** | Begrenzte Auswirkungen | Ueberschaubar |
| **Hoch** | Betraechtliche Auswirkungen | Gross |
| **Sehr hoch** | Existenzbedrohende Auswirkungen | Katastrophal |

- **Schutzbedarfsvererbung**: Schutzbedarf vererbt sich von Geschaeftsprozessen auf unterstuetzende Assets
  - **Maximumprinzip**: Hoechster Schutzbedarf der verarbeiteten Daten bestimmt
  - **Kumulation**: Viele Systeme mit normalem Schutzbedarf koennen in Summe hoch sein
  - **Verteilung**: Redundanz kann Schutzbedarf senken

### Phase 3: Modellierung
- **Bausteine zuordnen**: Jedes Zielobjekt erhaelt passende Bausteine
- **Reihenfolge**: Prozess-Bausteine vor System-Bausteinen
- **Gruppenmodellierung**: Gleichartige Objekte einmal modellieren

### Phase 4: IT-Grundschutz-Check (Soll/Ist-Vergleich)
- Jede Anforderung pruefen: umgesetzt / teilweise / nicht umgesetzt / entbehrlich
- Dokumentation der Umsetzung
- Identifikation von Defiziten

### Phase 5: Risikoanalyse (nur bei hohem/sehr hohem Schutzbedarf)
- Verweis auf BSI 200-3
- Ergaenzende Sicherheitsanalyse
- Zusaetzliche Massnahmen ableiten

### Phase 6: Realisierung
- **Realisierungsplan**: Priorisierung, Zeitplan, Verantwortlichkeiten
- **Reihenfolge**: Basis -> Standard -> Hoch
- **Budget- und Ressourcenplanung**
- **Umsetzungsbegleitende Dokumentation**
