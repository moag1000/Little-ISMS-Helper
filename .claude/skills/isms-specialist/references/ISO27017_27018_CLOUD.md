# ISO/IEC 27017 & 27018 — Cloud Security & Privacy

## ISO/IEC 27017:2015 — Cloud Security Controls

### Ueberblick
Umsetzungshinweise fuer cloud-spezifische Informationssicherheitskontrollen. Ergaenzt ISO 27002 um Cloud-spezifische Aspekte.

### Zusaetzliche Cloud Controls (ueber ISO 27002 hinaus)

| Control | Titel | Cloud-Aspekt |
|---------|-------|-------------|
| CLD.6.3 | Relationship between CSP and CSC | Verantwortlichkeitsabgrenzung |
| CLD.8.1 | Virtual machine hardening | VM-Isolation, Hypervisor-Sicherheit |
| CLD.9.5 | Virtual machine isolation | Netzwerk-/Speicher-Isolation |
| CLD.12.1 | Admin operations security | Cloud-Administrationssicherheit |
| CLD.12.4 | Clock synchronization in cloud | Zeitsynchronisation ueber Cloud-Infrastruktur |
| CLD.13.1 | Security in shared networks | Multi-Tenant-Netzwerktrennung |

### Erweiterte ISO 27002 Controls fuer Cloud

**Fuer Cloud Service Provider (CSP):**
- 5.1: Cloud-spezifische Security Policies
- 6.1: IS-Organisation mit Cloud-Governance
- 8.1: Asset-Management fuer Cloud-Ressourcen
- 9.2: Zugangssteuerung fuer Cloud-Management-Interfaces
- 9.4: Cloud-spezifische Zugangskontrollen (API-Keys, Tokens)
- 10.1: Kryptographie fuer Data-at-Rest und Data-in-Transit
- 11.2: Equipment-Sicherheit in Cloud-Rechenzentren
- 12.1: Betriebsverfahren fuer Cloud-Services
- 12.3: Kapazitaetsmanagement mit Elastizitaet
- 12.4: Logging und Monitoring in Cloud-Umgebungen
- 13.1: Netzwerksicherheit mit Virtualisierung
- 14.1: Sichere Entwicklung von Cloud-Services
- 15.1: Lieferantenbeziehungen (Sub-Prozessoren)
- 16.1: Incident Management in Cloud-Umgebungen
- 17.1: Business Continuity fuer Cloud-Services
- 18.1: Compliance-Anforderungen fuer Cloud

**Fuer Cloud Service Customer (CSC):**
- Verantwortlichkeiten des Kunden klar definiert
- Datentrennung und -isolation verifizieren
- Exit-Strategie und Datenportabilitaet sicherstellen
- Monitoring und Audit-Rechte vertraglich regeln

## ISO/IEC 27018:2019 — PII Protection in Public Cloud

### Ueberblick
Umsetzungshinweise zum Schutz personenbezogener Daten (PII) in oeffentlichen Cloud-Diensten. Ergaenzt ISO 27002 um Datenschutz-spezifische Cloud-Aspekte.

### Prinzipien

1. **Einwilligung und Wahlmoeglichkeit**: PII nur nach Weisung des Cloud-Kunden verarbeiten
2. **Zweckbindung**: Nur fuer vereinbarte Zwecke
3. **Datenminimierung**: Nicht mehr als notwendig
4. **Transparenz**: Offenlegung der Verarbeitung
5. **Zugangs- und Berichtigungsrechte**: Mechanismen fuer Betroffene
6. **Loeschung**: Nach Vertragsende/Anfrage

### Zusaetzliche PII-Controls

| Control | Titel | Anforderung |
|---------|-------|------------|
| A.1 | PII return/disposal | Rueckgabe/Loeschung nach Vertragsende |
| A.2 | Sub-processors | Offenlegung und Kontrolle |
| A.3 | PII disclosure notification | Meldung bei Offenlegung an Behoerden |
| A.4 | PII breach notification | Benachrichtigung bei Datenpanne |
| A.5 | Audit and compliance | Audit-Rechte des Cloud-Kunden |
| A.6 | Cooperation with PII principals | Unterstuetzung bei Betroffenenrechten |
| A.7 | Purpose restriction | Keine Verarbeitung fuer eigene Zwecke |
| A.8 | Disclosure | Transparenz ueber Jurisdiktionen |
| A.9 | Data location | Information ueber Speicherorte |
| A.10 | PII return/transfer | Sichere Rueckuebertragung |
| A.11 | Return/disposal confirmation | Nachweis der Loeschung |
